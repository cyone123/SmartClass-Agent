当前主链路已经很完整，但距离“企业级真实落地”主要差在产品化边界、账号权限、安全合规、稳定性、质量评测、运维观测和规模化部署。

## **一、总体目标**
将 SmartClass 从“功能主链路完整的教学 Agent 原型”升级为“可在学校、教培机构、教育平台中真实部署、可运维、可审计、可持续迭代的教师 AI 工作平台”。

核心升级方向：

- 从单教师备课工具升级为学校/教研组级协作平台
- 从功能可用升级为质量可控、过程可追踪、成本可管理
- 从本地演示系统升级为支持多租户、权限、部署、监控、审计的企业级系统
- 从“生成结果”升级为“教学资产生命周期管理”

## **二、产品需求完善**
1. 用户与组织体系

- 增加真实账号体系，替换前端 mock 账户
- 支持教师、教研组长、学校管理员、平台管理员等角色
- 支持学校、年级、学科、教研组、班级等组织结构
- 支持个人资源、教研组资源、学校公共资源的边界划分

2. 备课业务流程产品化

- 增加标准备课流程模板：新授课、复习课、实验课、公开课、作业讲评课
- 增加课程标准、教材版本、学段、地区、学情等结构化字段
- 增加“教学目标-活动设计-评价任务-作业设计”一致性检查
- 增加教案、课件、课堂互动、练习题、板书设计、教学反思的一体化生成

3. 协作与审核

- 支持教研组共同备课、评论、批注、版本对比
- 支持公开课/校本资源审核流
- 支持资源发布到校本资源库
- 支持优秀案例沉淀与复用

4. 教学资产管理

- 将 artifact 升级为正式教学资产
- 支持标签、收藏、复制、归档、分享、导出
- 支持版本对比、历史回滚、当前版本锁定
- 支持按学科、年级、教材、知识点检索资产

## **三、AI 能力增强**
1. 教学质量增强

- 建立教学设计质量 rubric：目标清晰度、活动合理性、重难点覆盖、评价一致性
- 增加生成前规划、生成后自检、用户确认后的修订闭环
- 引入“教学专家审阅 Agent”，专门检查教学逻辑与学科准确性
- 支持不同教学风格：探究式、项目式、讲授式、任务驱动式

2. RAG 与知识库增强

- 支持教材目录、课标、校本资料的结构化知识库
- 增加知识切片质量评估、重复检测、失效文档清理
- **增加引用来源展示，让教师知道内容依据来自哪个文件**
- 增加混合检索：关键词 + 向量 + 元数据过滤 + rerank

3. 多模态输入增强

- 继续完善图片资料解析、PPT 参考资料解析
- 支持教材页截图、手写教案、课堂板书图片识别
- 视频理解从“转写+关键帧摘要”升级为章节切分、片段定位、教学用途建议
- 增加解析结果结构化存储，而不仅是摘要文本

4. 产物生成增强

- PPT 增强：模板库、品牌风格、版式继承、动画建议、图表自动生成
- DOCX 增强：学校标准教案模板、页眉页脚、表格化教案、批注修订
- HTML 互动增强：选择题、拖拽匹配、课堂小游戏、实验模拟、即时反馈
- 增加 PDF、习题卷、课堂任务单、评价量表等 artifact 类型

## **四、技术架构完善**
1. 协议治理

- 固化 SSE 事件契约：`metadata/token/progress/artifact/artifact_trace/approval/error/done`
- 为 artifact、progress、approval、attachment analysis 建立统一 schema
- 前后端共享 TypeScript/OpenAPI 类型，减少字段漂移
- 增加协议版本号，兼容旧客户端

2. 后端工程化

- 增加任务队列：文件解析、视频分析、产物生成应从请求线程中解耦
- 增加任务状态机：pending/running/retry/success/failed/canceled
- 增加幂等控制，避免重复点击导致重复生成
- 增加后台任务恢复机制，服务重启后可继续或标记失败

3. 存储架构

- 本地文件存储升级为可插拔存储：本地、MinIO、S3、学校私有对象存储
- 数据库增加迁移管理，如 Alembic
- 明确 artifact、attachment、knowledge_file 的生命周期和清理策略
- 增加存储配额、文件大小限制、病毒扫描/内容安全扫描

4. 安全与权限

- 当前 [main.py](</d:/Learn/langchain/demo/backend/app/main.py>) 中 CORS 是 `allow_origins=["*"]`，生产环境需要收紧
- 增加登录认证、JWT/session、RBAC 权限
- 增加 plan/thread/file/artifact 的归属校验
- OnlyOffice 回调增加签名校验和来源校验
- Agent workspace 继续强化沙箱、超时、资源限制、审计记录

## **五、可观测性与运维**
1. 日志与链路追踪

- 统一结构化日志：`request_id/run_id/thread_id/plan_id/user_id/artifact_id`
- 将 artifact trace 与系统 trace 分层：前者给用户看，后者给运维排障
- 接入 OpenTelemetry，跟踪 API、LangGraph 节点、LLM 调用、RAG 检索、文件任务

2. 指标监控

- API 延迟、错误率、SSE 中断率
- LLM 调用次数、token 消耗、模型失败率、重试率
- RAG 命中率、入库耗时、切片数量
- artifact 生成成功率、平均耗时、失败原因分布
- 文件解析队列长度、任务积压、存储用量

3. 告警与运维面板

- 模型接口不可达告警
- OnlyOffice 回写失败告警
- 文件解析连续失败告警
- 生成任务超时告警
- 成本异常告警

## **六、质量保障**
1. 自动化测试

- 现有后端已有多条测试，如 [backend/tests](</d:/Learn/langchain/demo/backend/tests>)，建议补齐 API 集成测试、数据库迁移测试、失败路径测试
- 增加前端组件测试和 E2E 测试，覆盖聊天流、审批、文件上传、产物预览
- 增加 SSE 协议回归测试
- 增加 fixture 样例库：PDF、DOCX、PPT、视频、图片、异常文件

2. AI 评测体系

- 建立标准教学任务集
- 对意图识别、元数据抽取、RAG 引用、教学设计、PPT 质量分别评分
- 增加人工评审后台，收集教师评分
- 将失败案例沉淀为回归评测集

## **七、部署交付**
- 提供 Docker Compose：backend、frontend、PostgreSQL、OnlyOffice、对象存储
- 提供 Windows 本地部署包和 Linux 服务器部署方案
- 增加 `.env.example`、初始化脚本、健康检查接口
- 增加 CI/CD：lint、test、build、镜像构建、迁移检查
- 增加灰度发布、数据备份、恢复方案

## **八、建议路线图**
第一阶段：产品化补短板，2-4 周

- 替换 mock 账户，建立认证、用户、角色、组织
- 收紧 CORS 与文件访问权限
- 固化 SSE/artifact/progress/approval 协议
- 增加基础部署文档、`.env.example`、健康检查

第二阶段：企业级稳定性，4-8 周

- 引入任务队列、结构化日志、指标监控、OpenTelemetry
- 完善对象存储、数据库迁移、任务恢复、失败重试
- 建立自动化测试和 AI 评测集
- 增强 OnlyOffice 回写安全与文件生命周期管理

第三阶段：教育场景深化，8-12 周

- 增强图片/PPT解析、教材/课标结构化知识库
- 增加教研协作、资源审核、资产库
- 增加模板库、风格继承、教学质量自检
- 扩展习题、任务单、评价量表等产物类型

总体判断：这个项目最值得保留和继续放大的核心资产，是 `LangGraph 工作流 + 多模态上下文 + artifact 版本链 + OnlyOffice 回写 + 过程可视化`。下一步不要大拆，而是围绕企业级落地补“身份、权限、任务、观测、评测、部署、资产治理”这几根承重梁。
